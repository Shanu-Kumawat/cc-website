## Rules and Guidelines

1. **Code of Conduct**: Be respectful, inclusive, and supportive. Harassment of any kind will not be tolerated.

2. **Original Work**: All code must be written during the hackathon. You can use libraries, frameworks, and APIs, but the core project must be new.

3. **Pre-existing Code**: You can use boilerplate/starter code, but must clearly state what was pre-written.

4. **Team Size**: 2-4 members per team. Solo participants are allowed but encouraged to find teammates.

5. **Submission**: Projects must be submitted via GitHub by the code freeze deadline.

6. **Presentation**: All team members should participate in the presentation.

7. **Open Source**: Projects should be open-sourced on GitHub (private during hackathon, public after).

8. **IP Rights**: Participants retain full ownership of their projects.

## FAQs

**Q: I'm a beginner. Can I participate?**  
A: Absolutely! We encourage beginners. You'll have mentors to guide you, and it's a great learning experience.

**Q: Do I need to stay for the entire 48 hours?**  
A: You can come and go, but we recommend staying overnight for the full experience. Sleeping areas are available.

**Q: What tech stack should I use?**  
A: Any! Use whatever you're comfortable with. Popular choices: React/Node.js, Python/Flask, Flutter, etc.

**Q: Can I work on an existing project?**  
A: No. The project must be built from scratch during the hackathon. You can use existing libraries/frameworks.

**Q: What if I don't have a team?**  
A: Come solo! We have a team formation session where you can find teammates with complementary skills.

**Q: Is food really free?**  
A: Yes! All meals, snacks, and beverages are completely free for registered participants.

**Q: What happens if my laptop dies?**  
A: We have a few backup laptops available on a first-come basis. Make sure yours is in good condition!

**Q: Can I leave and come back?**  
A: Yes, but you must sign in/out for security. Keep your participant badge at all times.
